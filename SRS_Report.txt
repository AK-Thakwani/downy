1. INTRODUCTION & PROBLEM STATEMENT
-----------------------------------
Our project, DownyShoes, was built to solve the mess of old e-commerce sites that are hard to update. Most basic sites use "spaghetti code" where everything is in one file. We decided to use the MVC (Model-View-Controller) design pattern to keep our database, our logic, and our UI separate. This makes the site way faster and easier to fix if something breaks.

2. TECHNICAL FOCUS: DESIGN PATTERNS (MVC)
-----------------------------------------
We focused heavily on refactoring our code into a clean MVC structure.
- MODELS: We made custom models for Product, Cart, and Order. All SQL queries stay here.
- VIEWS: We used Handlebars (hbs) for the front-end to keep it dynamic and modern.
- CONTROLLERS: This is where we put the "brains" of the site, like checking if a shoe is in stock before letting someone buy it.

3. SYSTEM DIAGRAMS (LOGIC)
--------------------------
USE CASE SUMMARY:
- Customer: Browse Shoes, Manage Cart, Guest Checkout, Track Order Status.
- Admin: View Sales Stats, Update Inventory, Manage Customer Records.

MVC ARCHITECTURE LOGIC:
- Controller --> ProductModel (To Fetch Shoe Data)
- Controller --> OrderModel (To Save Sales and check Revenue)
- Controller --> View Templates (To send the final design to the user's screen)

4. MAIN FEATURES (FUNCTIONAL REQUIREMENTS)
------------------------------------------
- Interactive Shop: Users can sort shoes by price or name easily.
- Stock Guard: The system won't let you add a shoe to the cart if the quantity is 0 in the database.
- Live Admin Panel: Admin can log in (using admin / 7258) to see a "Sales Pulse" chart and total revenue.
- Order Tracking: After buying, you get a unique Order ID to check if your shoes are "Pending" or "Shipped" on the /track page.

5. NON-FUNCTIONAL REQUIREMENTS
------------------------------
- Design: We used a Neon Green and Beige theme to make it look like a high-end brand.
- Speed: We optimized the MySQL queries so the inventory updates and search results are instant.
- Security: Admin pages are protected by session checks so customers cannot access the private data.

6. PERFORMANCE & VALIDATION
---------------------------
We tested the system by adding 35+ products and doing multiple fake checkouts. We compared the MVC structure to our original code and found that finding bugs is now much faster because the code is organized into folders. We also successfully handled "out of stock" edge cases so the site never crashes.
